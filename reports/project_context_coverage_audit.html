路径散落检查结果

文件结果建议
app.py只保留说明文案中的 outputs、project_config.json，以及业务变量 project_config。后续拆分 UI 时再继续降低 app 体积。

project_context.py集中保存路径常量、标题清洗和轻量 path helper。保持只做路径，不增加业务扫描、导出或生成逻辑。

本次小范围收口

  • 新增 get_outputs_root()，让输出根目录入口也收敛到 project_context.py。
  • file_manager.ensure_directories() 和 list_project_titles() 不再直接读取 OUTPUT_DIR。
  • save_project_config() 改为使用 ctx.config_path。
  • app.py 移除对 OUTPUT_DIR 和 get_project_dir() 的直接依赖。

职责边界评价

ProjectContext 未发现越界。 当前职责集中在标题清洗、项目路径、常用文件路径、章节/摘要 path helper 和创建当前必需目录。它没有保存章节、生成索引、调用模型、拼 Prompt、读取 UI 状态或管理导出内容。

file_manager.py 边界基本清晰，仍承担文件读写和版本选择职责。app.py 仍偏大，但本次不建议做完整拆分。

下一步建议

  • 可以继续让新增文件能力优先依赖 ProjectContext，避免再次散落路径拼接。
  • 可以准备 workspace/books 迁移设计，但不建议现在直接迁移目录。
  • 不建议现在做 book_id、book.json、revisions、exports/logs 目录体系或数据库。
